@AGENTS.md

This is an Expo SDK 56 app (React Native 0.85, React 19) using Expo Router, TypeScript (strict), the React Compiler, and typed routes. It is a demo **habit tracker**.

## Commands

```bash
npm start              # Metro dev server (interactive: i=iOS, a=Android, w=web, r=reload)
npm run ios            # start + open iOS simulator
npm run web            # start + open web
npx tsc --noEmit       # type-check (run this to verify changes; there is no test suite)
npm run lint           # expo lint (ESLint)
npx expo-doctor        # validate project/native config
npm run reset-project  # ⚠️ moves current app to app-example/ and scaffolds a blank app — do not run casually
```

There is no test framework configured. `tsc --noEmit` is the primary correctness gate.

## Architecture

**Source lives in `src/`** (not the repo root). Expo Router's route root is configured to `src/app`. Path aliases: `@/*` → `src/*`, `@/assets/*` → `assets/*`.

**Navigation is unusual — read before touching it.** `src/app/_layout.tsx` does *not* use Expo Router's `Tabs`. It renders `ThemeProvider` → `HabitsProvider` → `<AppTabs />`. The tab bar lives in `src/components/app-tabs.tsx` using `NativeTabs` from `expo-router/unstable-native-tabs`. Tab **labels** are set there (Today / Progress), while the route files keep their original filenames (`index.tsx`, `explore.tsx`). To add a screen you wire both the route file under `src/app/` and a `NativeTabs.Trigger` in `app-tabs.tsx`.

**Styling goes through theme primitives, not raw `Text`/`View`.** `src/constants/theme.ts` defines `Colors` (light/dark), a `Spacing` scale (`half`…`six`), `Fonts`, `BottomTabInset`, and `MaxContentWidth`. `useTheme()` (`src/hooks/use-theme.ts`) resolves the active scheme. Use `ThemedText` and `ThemedView` (`src/components/`):
- `ThemedText` `type` is a fixed union: `default | title | small | smallBold | subtitle | link | linkPrimary | code`. **There is no `defaultSemiBold`** — apply font weight via `style`.
- `ThemedView` `type` is a `ThemeColor` key (e.g. `backgroundElement`) selecting its background.

**Domain logic is in `src/lib/`:**
- `habits-store.tsx` — `HabitsProvider` (React context) + `useHabits()` hook. Single source of truth for habits; persists the whole array to AsyncStorage key `habits.v1` on every change and seeds 3 default habits on first load. A `Habit.history` is a `Record<'YYYY-MM-DD', boolean>` of completed days.
- `date.ts` — local-time date keys plus `currentStreak` / `longestStreak` / `lastNDays` / `weekdayLabel`. Streak logic counts back from today; an incomplete today does not break the streak (counting starts from yesterday).

Screens: `src/app/index.tsx` is **Today** (toggle completion, add/delete habits, daily progress ring); `src/app/explore.tsx` is **Progress** (last-7-days grid, streak/completion stats).

**Web vs native:** some components ship a parallel `*.web.tsx` (e.g. `animated-icon`, `app-tabs`, `use-color-scheme`); Metro picks the platform variant automatically. CSS (`global.css`, `*.module.css`) is web-only — its module types come from `expo-env.d.ts` (auto-generated, gitignored; regenerated by `expo start`).

## Environment notes

- **iOS simulator works** (Xcode present). **Android is not set up** (needs JDK 17+ / Android Studio); use Expo Go or EAS for Android.
- AsyncStorage is bundled into Expo Go, so the app runs there without a custom dev build.
